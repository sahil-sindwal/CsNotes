Three Schema Architecture

The three schema architecture is also called ANSI/SPARC architecture or three-level architecture. This framework is used to describe the structure of a specific database system. The three schema architecture is also used to separate the user applications and physical database. The three schema architecture contains three-levels. It breaks the database down into three different categories.

The main objective of three level architecture is to enable multiple users to access the same data with a personalized view while storing the underlying data only once. Thus it separates the user's view from the physical structure of the database. This separation is desirable for the following reasons:

  • Different users need different views of the same data.
  • The approach in which a particular user needs to see the data may change over time.
  • All users should be able to access the same data according to their requirements.

1) The Internal Level

It is the one closest to physical storage. The internal level has an internal schema which describes the physical storage structure of the database. It is also known as physical schema. It uses the physical data model. It is used to define that how the data will be stored in a block. The following aspects are considered at this level:

  • Storage space allocation for data
  • record description for storage with stored sizes for data items.
  • Data compression and encryption techniques.
  • Optimization of the internal structures.

2) Conceputal Level

This level is the representation of the entire contents of data base. It is defined by conceputal schema which includes definations of each of the various types of data or the various types of conceputal record. It hides the details of physical storage structures and components on describing entities, data types, relationship, user operations and constraints. It create many external views, each consisting more or less abstract representation of some portion of data base and there will be a single conceptual veiw. It is also known as logical level. This level is concenred with following activities:

  • all entities, their attributes and their relationship.
  • Constraints on the data.
  • Security and integrity rules.
  • Semantic information about the data.

3) External Level

This level is one closed to the users. This level is concenred with the way in which the data is viewed by individual users. At external level, the dbms presents each user with a shared single view or schema of the data. There are many view at this level, and each view is representation of the total database. A view allow user to access the portion of database from them. It is defined by external schema which consist basiclly of definations of each of the various types record found n the external view.

Advantages of three level architecture

  • Each user is able to access the same data but have a different customized view of the data as per their own needs.
  • A user can change his view and this change doesn’t affect other user’s.
  • User’s interaction wit database is independent of physical data storage organization.

Mapping Between Views

The three levels of DBMS architecture don't exist independently of each other. There must be correspondence between the three levels i.e. how they actually correspond with each other. DBMS is responsible for correspondence between the three types of schema. This correspondence is called Mapping. There are basically two types of mapping in the database architecture:

  • Conceptual/ Internal Mapping
  • Conceptual/ Internal Mapping

Conceptual/ Internal Mapping The Conceptual/ Internal Mapping lies between the conceptual level and the internal level. Its role is to define the correspondence between the records and fields of the conceptual level and files and data structures of the internal level.

External/ Conceptual Mapping: The external/Conceptual Mapping lies between the external level and the Conceptual level. Its role is to define the correspondence between a particular external and the conceptual view.
